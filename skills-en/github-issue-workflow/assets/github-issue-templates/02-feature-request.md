## Summary

What should be added or improved?

## Business Goal

What user need or business outcome does this support?

## Problem

What limitation or gap does this address?

## Desired Outcome

What should be true after this lands?

## Constraints

What technical, product, compatibility, or schedule constraints apply?

## Dependencies

What upstream work, approvals, data, or services must be in place first?

## Impacted Files or Systems

Which files, modules, services, configs, or migrations are likely to change?

## Rollout / Migration Notes

How should this be introduced safely if it affects users, data, or interfaces?

## Alternatives

What other approaches were considered?

## Risks

What could go wrong or increase complexity?

## Acceptance Criteria

- [ ] Scope is clear
- [ ] Dependencies are identified
- [ ] Implementation path is understood
- [ ] Verification is defined
- [ ] Outcome can be verified
