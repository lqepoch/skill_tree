## 摘要

想增加或改进什么？

## 业务目标

这个需求要支持什么用户目标或业务结果？

## 问题

它解决了什么限制或缺口？

## 目标结果

落地后应该达到什么状态？

## 约束

有哪些技术、产品、兼容性或时间上的约束？

## 依赖

需要哪些上游工作、审批、数据或服务先准备好？

## 受影响文件或系统

可能会改动哪些文件、模块、服务、配置或迁移？

## 发布 / 迁移说明

如果影响用户、数据或接口，应该如何安全地引入？

## 备选方案

还考虑过什么其他做法？

## 风险

可能会出现什么问题，或者哪里会增加复杂度？

## 验收标准

- [ ] 范围已明确
- [ ] 依赖已识别
- [ ] 实现路径已明确
- [ ] 验证方式已定义
- [ ] 结果可以被验证
