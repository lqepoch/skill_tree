# 功能需求

> 使用说明：功能需求要写清楚“为什么需要”、“要解决什么问题”、“什么算成功”，并明确非目标，避免 scope 膨胀。

## 1. 摘要

- 需求一句话说明：
- 主要受益对象：
- 目标结果：

### 示例

> 为后台运营增加批量导出订单 CSV 的能力，减少人工逐条下载订单的时间。

## 2. 问题与业务目标

请说明这个需求为什么存在。

- 当前痛点是什么：
- 业务目标是什么：
- 如果不做会怎样：
- 这个需求对用户、团队或流程的价值：

### 示例

> 运营每周需要导出数百条订单，目前只能逐页翻找，平均耗时 2 小时，容易漏单。

## 3. 期望结果

写出上线后应该看到的结果。

- 用户能做什么：
- 系统会发生什么变化：
- 数据、界面或接口会如何变化：

### 示例

> 运营可以在订单列表选择日期范围后，一键下载 CSV 文件，文件包含订单号、金额、状态和创建时间。

## 4. 非目标

明确这个需求不做什么，防止范围扩大。

- 不包含什么：
- 暂不支持什么：
- 后续再考虑什么：

### 示例

> 本次只支持 CSV，不支持 Excel 模板定制；不做历史数据清洗。

## 5. 约束与依赖

请把实施约束写清楚。

- 技术约束：
- 产品约束：
- 合规或安全约束：
- 外部依赖：
- 时间窗口或发布窗口：

### 示例

- 导出文件不能超过 10MB
- 必须保留权限校验
- 依赖现有异步任务服务

## 6. 影响范围

列出可能改动的文件、模块或系统。

- 文件：
- 模块：
- 服务：
- API：
- 数据库或迁移：

### 示例

- `app/admin/orders/page.tsx`
- `server/export/orders.ts`
- `jobs/export-orders-worker.ts`

## 7. 用户流程 / 交互

如果功能涉及 UI 或 API，请描述关键流程。

- 入口在哪里：
- 用户如何触发：
- 成功后看到什么：
- 失败时如何提示：

### 示例

1. 打开订单列表
2. 选择日期范围
3. 点击“导出 CSV”
4. 系统生成文件并提供下载链接

## 8. 方案与备选方案

给出可行实现路径，并说明推荐哪一个。

- 方案 A：
- 方案 B：
- 推荐方案：
- 原因：
- 风险：

### 示例

- 方案 A：同步生成后直接下载
- 方案 B：异步任务生成并通知下载
- 推荐方案：方案 B，因为它可以更好地处理大数据量

## 9. 发布 / 迁移说明

如果功能影响现有用户、接口或数据，请写清楚。

- 是否需要灰度：
- 是否需要迁移旧数据：
- 是否需要向后兼容：
- 是否需要通知用户：

### 示例

> 对现有接口保持兼容，新字段只做追加，不修改旧响应结构。

## 10. 验证计划

写出验收前的验证方式。

- 单元测试：
- 集成测试：
- 手工验证：
- 回归检查：

### 示例

- 校验 CSV 行数与筛选条件一致
- 验证无权限用户无法导出
- 验证大数据量任务会进入排队状态

## 11. 接受标准

- [ ] 业务目标已明确
- [ ] 非目标已明确
- [ ] 约束和依赖已确认
- [ ] 实现路径已确定
- [ ] 验证方式已定义
- [ ] 该需求可以进入开发或评审

## 12. 示例成品

### 示例标题

`[feature] Add CSV export for order list`

### 示例正文

- 摘要：为订单列表增加 CSV 导出
- 业务目标：减少运营人工导出耗时
- 期望结果：可按日期范围批量导出订单
- 非目标：本期不支持 Excel 格式
- 约束：导出必须复用权限校验，文件大小不超过 10MB
- 影响范围：前端列表页、导出服务、任务队列
- 验证计划：权限验证、导出字段校验、大数据量场景测试
- 接受标准：导出结果字段完整且与筛选条件一致
