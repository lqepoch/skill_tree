# 任务

> 使用说明：任务类 issue 应该非常具体，明确交付物、边界、验证方式和后续处理，避免写成笼统的“做一下优化”。

## 1. 摘要

- 任务一句话说明：
- 交付物是什么：
- 完成后应达到什么状态：

### 示例

> 清理订单服务中未使用的实验代码，并补上对应说明文档。

## 2. 背景

说明为什么需要这个任务。

- 触发原因：
- 当前痛点：
- 相关决策或上下文：

### 示例

> 旧实验代码影响阅读和排障，且已经不再被任何流量使用。

## 3. 范围

明确做什么，不做什么。

- 包含：
- 不包含：
- 边界条件：

### 示例

- 包含：删除未使用的 feature flag 分支
- 不包含：重构订单核心逻辑

## 4. 可能涉及的文件 / 系统

列出你预估会改动的地方。

- 文件：
- 模块：
- 服务：
- 配置：

### 示例

- `src/orders/legacy.ts`
- `docs/orders-maintenance.md`

## 5. 执行计划

把任务拆成可执行步骤。

1. ...
2. ...
3. ...

### 示例

1. 确认没有运行时依赖
2. 删除旧分支和无效配置
3. 更新文档说明

## 6. 验证步骤

说明怎样确认任务完成。

- 自动化检查：
- 手工检查：
- 回归检查：

### 示例

- 运行相关测试
- 确认构建通过
- 搜索仓库中不再引用旧代码路径

## 7. 后续 / 回滚

如果任务涉及风险，请说明后续处理。

- 是否需要回滚路径：
- 是否需要补充说明：
- 是否还会有后续拆分任务：

### 示例

> 若删除后发现还有未发现的调用点，先恢复文件，再补充引用扫描。

## 8. 接受标准

- [ ] 任务范围已清楚
- [ ] 交付物已完成
- [ ] 验证已通过
- [ ] 后续事项已记录

## 9. 示例成品

### 示例标题

`[task] Remove unused order feature flag code`

### 示例正文

- 摘要：移除未使用的订单实验代码
- 背景：该代码已不再受流量使用，增加维护成本
- 范围：仅删除未使用分支和相关配置，不修改核心流程
- 可能涉及：`src/orders/legacy.ts`、`docs/orders-maintenance.md`
- 执行计划：确认无引用 -> 删除 -> 更新文档 -> 验证构建
- 验证：测试通过、全局搜索无残留引用
- 后续：若发现遗漏引用，恢复并补充清理
